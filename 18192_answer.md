### 1. 应用 Master 方法求解以下递归方程（10分）

#### (1) $T(n) = 27T(n/3) + 11n^3$

**▶ 题干：**

用 Master 方法解递归方程 $T(n) = 27T(n/3) + 11n^3$。

**▶ 通俗解释：**

回顾 Master 定理——它就像解递归方程的"公式计算器"。对于形如 $T(n) = aT(n/b) + f(n)$ 的递归式，我们比较子问题权重 $n^{\log_b a}$ 和合并开销 $f(n)$ 谁大谁小。

本题中 $a=27$ 个子问题，每个缩小到 $1/3$。系数 11 在渐近分析中不重要（直接扔掉）。

**▶ 答案：**

**Step 1：提取参数**
- $a = 27$, $b = 3$
- $f(n) = 11n^3$

**Step 2：计算 $\log_b a$**
$$\log_b a = \log_3 27 = \log_3 3^3 = 3$$

所以 $n^{\log_b a} = n^3$。

**Step 3：比较 $f(n)$ 与 $n^{\log_b a}$**
$$f(n) = 11n^3 = \Theta(n^3) = \Theta(n^{\log_b a})$$

$f(n)$ 与 $n^{\log_b a}$ 同阶（都是 $n^3$），符合 **Case 2**（$k = 0$）。

**Step 4：套公式**
$$T(n) = \Theta(n^{\log_b a} \cdot \lg n) = \Theta(n^3 \lg n)$$

$$\boxed{T(n) = \Theta(n^3 \lg n)}$$

---

#### (2) $T(n) = 64T(n/4) + 10n^3 \log_2 n$

**▶ 答案：**

**Step 1：提取参数**
- $a = 64$, $b = 4$
- $f(n) = 10n^3 \log_2 n$

**Step 2：计算 $\log_b a$**
$$\log_b a = \log_4 64 = 3$$

因为 $64 = 4^3$。所以 $n^{\log_b a} = n^3$。

**Step 3：比较**
$$f(n) = 10n^3 \lg n = \Theta(n^3 \lg n) = \Theta\left(n^{\log_b a} \cdot \lg^1 n\right)$$

符合 **Case 2 扩展形式**：$f(n) = \Theta(n^{\log_b a} \cdot \lg^k n)$，其中 $k = 1$。

**Step 4：套公式**
$$T(n) = \Theta\left(n^{\log_b a} \cdot \lg^{k+1} n\right) = \Theta(n^3 \lg^2 n)$$

$$\boxed{T(n) = \Theta(n^3 \lg^2 n)}$$

> **对比两小题**：(1) 的 f(n) 没有对数因子，结果是 $\Theta(n^3 \lg n)$；(2) 的 f(n) 多了一个 $\lg n$，所以结果也多一次 $\lg n$ → $\Theta(n^3 \lg^2 n)$。多一个 log，结果也多一个 log。

**▶ 举一反三——新例题：**

**【新题】** 用 Master 方法解 $T(n) = 8T(n/2) + n^2$。

**【答案】** $\Theta(n^3)$。
- $a=8, b=2, \log_2 8 = 3, n^{\log_b a} = n^3$
- $f(n) = n^2 = O(n^{3-1})$（$\varepsilon = 1$），符合 Case 1
- 子问题权重 $n^3$ 主导，淹没合并开销 $n^2$

**【简略解析】** Case 1 的关键词是"子问题主导"——合并开销根本追不上子问题增长。想象你每个子问题都很重（$n^3$ 级别），而合并只要 $n^2$，那就看子问题就行了。

### 2. 展开递归式 $T(n) = T(n-1) + cn$ 并做渐近分析（5分）

**▶ 题干：**

展开递归式 $T(n) = T(n-1) + cn$，并做渐近分析。

**▶ 通俗解释：**

这不是"分治"型递归（因为没有 $T(n/b)$ 这种除法），而是"**递减型**"递归——每次规模只减少 1。这种递归不能直接用 Master 定理，需要自己**展开**（也叫迭代法/代入法）。

可以理解为：第 n 步的代价 = 第 n-1 步的代价 + 当前步的额外开销 $cn$。展开就是一层层剥开，看总代价。

**▶ 答案：**

**递推展开：**

$$\begin{aligned}
T(n) &= T(n-1) + cn \\
     &= [T(n-2) + c(n-1)] + cn \\
     &= T(n-2) + c(n-1) + cn \\
     &= [T(n-3) + c(n-2)] + c(n-1) + cn \\
     &= T(n-3) + c[(n-2) + (n-1) + n] \\
     &= \cdots \\
     &= T(0) + c(1 + 2 + \cdots + n) \quad \text{（假设递推到 } T(0) \text{）}
\end{aligned}$$

**等差数列求和：**
$$1 + 2 + \cdots + n = \frac{n(n+1)}{2}$$

**代入：**
$$T(n) = T(0) + c \cdot \frac{n(n+1)}{2} = T(0) + \frac{c}{2}n^2 + \frac{c}{2}n$$

**渐近分析：**
$$\boxed{T(n) = \Theta(n^2)}$$

（常数项 $T(0)$ 和一次项 $\frac{c}{2}n$ 在渐近意义下可忽略，最高次项 $n^2$ 主导增长率。）

> **记忆技巧**：$T(n) = T(n-1) + f(n)$ 展开后就是 $\sum_{i=1}^{n} f(i)$。本题 $f(i) = ci$，求和得 $\Theta(n^2)$。

**▶ 举一反三——新例题：**

**【新题 1】** 展开 $T(n) = T(n-1) + c$（c 为常数），求渐近复杂度。

**【答案】** $T(n) = T(0) + cn = \Theta(n)$。

**【新题 2】** 展开 $T(n) = T(n-2) + n$（假设 $T(1)=1$），求渐近复杂度。

**【答案】** $\Theta(n^2)$。展开约 $n/2$ 项，每项约 $n$ 级别，总和仍是平方级。（系数只是差个常数因子）

**【简略解析】** 递推展开的精髓：找到第 k 次展开后的形式，归纳出通项，然后对各项求和。

### 3. 分析插入算法在最好、最坏和平均情形的关键字比较次数（10分）

**▶ 题干：**

```text
For (i ← n-1; i ≥ 0 且 x < a[i]; i--)
    a[i+1] ← a[i];
a[i+1] ← x;
```

将 x 插入到前 n 个元素已排好序的数组 $a[0:n]$ 中，分析算法在最好、最坏和平均情形所用的**关键字比较次数**。

**▶ 通俗解释：**

这道题和 21221 试卷的"二.1"是同一题（只是写成了伪代码形式）。核心操作是：把新元素 x 插入到一个**已排好序**的数组中，保持有序。

做法是从右往左扫，遇到比 x 大的元素就往后挪一格（腾位置），直到找到 x 应该待的位置。

本题明确要求统计的是**"关键字比较次数"**，即 `x < a[i]` 这个比较执行了多少次。

> 想象你有一叠从小到大排好的扑克牌，新抽了一张，你从右边开始往左比对，比自己大的牌往右移，找到合适位置就塞进去。

**▶ 答案：**

**最好情形：$\Theta(1)$**（仅 1 次比较）

- **条件**：$x \ge a[n-1]$（x 比数组中所有元素都大或等于最大的那个）
- **过程**：第一次循环条件判断 `i ≥ 0` 为真（`i = n-1 ≥ 0`），但 `x < a[n-1]` 为假。循环体一次也不执行，循环退出。
- **比较次数**：1 次（只比较了一次 `x < a[n-1]`）

---

**最坏情形：$\Theta(n)$**（n+1 次比较）

- **条件**：$x < a[0]$（x 比数组中所有元素都小）
- **过程**：循环体中移位操作执行 n 次（$i = n-1, n-2, \ldots, 0$），每次都要做 `x < a[i]` 的比较。最后 $i = -1$ 时还要做一次 `i ≥ 0` 的判断退出循环。
- **比较次数**：$n + 1$ 次（n 次 `x < a[i]` 为真，第 n+1 次 $i=-1$ 时 `i ≥ 0` 为假退出）

---

**平均情形：$\Theta(n)$**

- **条件**：x 的值在数组已排序元素的范围内均匀随机分布
- **分析**：x 的期望插入位置在所有可能位置的中间附近，大约有 $n/2$ 个元素需要比较和移位
- **比较次数**：约 $(n+1)/2$ 次
- **复杂度**：$\Theta(n)$（常数因子 1/2 在渐近意义下被忽略）

---

**总结表：**

| 情形 | 条件 | 比较次数 | 复杂度 |
|------|------|----------|--------|
| 最好 | $x \ge a[n-1]$ | 1 | $\Theta(1)$ |
| 最坏 | $x < a[0]$ | $n+1$ | $\Theta(n)$ |
| 平均 | x 随机 | $\approx \frac{n+1}{2}$ | $\Theta(n)$ |

**▶ 举一反三——新例题：**

**【新题】** 在长度为 n 的有序数组中，用**二分查找**查找元素 x。分析最好、最坏和平均情况下的比较次数。

**【答案】**

| 情形 | 条件 | 比较次数 | 复杂度 |
|------|------|----------|--------|
| 最好 | x = a[mid] 第一次就命中 | 1 | $\Theta(1)$ |
| 最坏 | x 不在数组中 | $\approx \lg n$ | $\Theta(\lg n)$ |
| 平均 | x 随机 | $\approx \lg n$ | $\Theta(\lg n)$ |

**【简略解析】** 二分查找每次砍掉一半搜索空间，所以即使最坏也就砍 $\lg n$ 次。注意和本题的对比：本题是**顺序方式**（一个个扫），复杂度 $\Theta(n)$；二分查找是**跳跃方式**（对半砍），复杂度 $\Theta(\lg n)$——但前提是数组**随机访问**（数组支持，链表就不行）。

### 4. 用步计数法分析 Rank 算法的渐近时间复杂度（5分）

**▶ 题干：**

```text
for (i ← 0; i < n; i++)
    r[i] ← 0;                         // 第一段：初始化
for (i ← 1; i < n; i++)               // 第二段：计算 rank
    for (j ← 0; j < i; j++)
        if (a[j] ≤ a[i]) r[i]++;
        else r[j]++;
```

分析该计算数组元素 rank 值算法的渐近时间复杂度。

**▶ 通俗解释：**

**Rank（排名/秩）**：一个元素在数组中的 rank 就是"有多少个元素不超过它"（即它在排序后应该排第几，从 0 开始计数）。比如数组 $[5, 2, 8]$ 中，5 的 rank 是 1（因为 2 ≤ 5，所以有一个不超过 5）。

这段代码的做法是**两两比较**：对每一对 $(i, j)$（$j < i$），看谁大，给大的 rank 加 1。这是一种"暴力比较"的方法。

**步计数法**就是看最内层的基本操作执行了多少次。这里的基本操作是 `if (a[j] ≤ a[i])` 的比较。

**▶ 答案：**

**第一段：初始化**
```text
for (i ← 0; i < n; i++)
    r[i] ← 0;
```
执行 $n$ 次赋值 → $\Theta(n)$。

**第二段：比较计数**
```text
for (i ← 1; i < n; i++)       // 外层循环 n-1 次
    for (j ← 0; j < i; j++)   // 内层循环 i 次
        if (...) ...           // 比较操作
```

关键：内层循环 $j$ 从 $0$ 到 $i-1$，执行 $i$ 次。

$$\text{总比较次数} = \sum_{i=1}^{n-1} i = \frac{(n-1) \cdot n}{2}$$

**渐近分析：**
$$\frac{n(n-1)}{2} = \Theta(n^2)$$

**总时间复杂度：**
$$\boxed{\Theta(n^2)}$$

（初始化 $\Theta(n)$ + 比较 $\Theta(n^2)$ = $\Theta(n^2)$，由高阶项主导。）

> **直观理解**：n 个元素两两比较，共有 $C_n^2 = \frac{n(n-1)}{2}$ 对——恰好是 $\Theta(n^2)$。

**▶ 举一反三——新例题：**

**【新题】** 用步计数法分析以下嵌套循环的渐近复杂度：

```c
for (i = 0; i < n; i++)
    for (j = 0; j < n; j++)
        x = x + 1;
```

```c
for (i = 0; i < n; i++)
    for (j = i; j < n; j++)
        x = x + 1;
```

**【答案】**

第一段：内层每次执行 $n$ 次，外层 $n$ 次 → $n \times n = n^2$ → $\Theta(n^2)$

第二段：内层从 $i$ 到 $n-1$，执行 $n-i$ 次：
$$\sum_{i=0}^{n-1} (n-i) = n + (n-1) + \cdots + 1 = \frac{n(n+1)}{2} = \Theta(n^2)$$

**【简略解析】** 两段都是 $\Theta(n^2)$，只是常数因子不同（$n^2$ vs $n^2/2$）。渐近分析中常数因子不重要——所以都是 $\Theta(n^2)$。

### 1. 叙述分治法算法的思想，并用归并排序算法说明（5分）

**▶ 题干：**

叙述分治法算法的思想，并用归并排序算法说明。

**▶ 通俗解释：**

**分治法（Divide and Conquer）** 就是"大事化小，小事化了，再汇总"：

1. **分（Divide）**：把大问题切成几个小问题（结构相同，规模更小）
2. **治（Conquer）**：递归地解决小问题（小到一定程度直接解决）
3. **合（Combine）**：把小问题的解合并成原问题的解

> 就像期末考试复习：你把一整本书撕成几章，每章分别复习，再把每章的笔记综合成一份总复习提纲。

**▶ 答案：**

**分治法三步骤：**

```
┌──────────────────────────────────────────────────┐
│  Divide（分解）                                   │
│  将原问题分解为若干个规模较小的、与原问题形式     │
│  相同的子问题                                     │
│                    ↓                              │
│  Conquer（求解）                                  │
│  递归地求解各个子问题（当子问题规模足够小时，     │
│  直接求解——这是递归的"基"）                      │
│                    ↓                              │
│  Combine（合并）                                  │
│  将各子问题的解合并为原问题的解                   │
└──────────────────────────────────────────────────┘
```

**用归并排序说明：**

**归并排序（Merge Sort）** 是最经典的分治法例子：

| 步骤 | 归并排序中的体现 |
|------|-----------------|
| **Divide** | 把数组从中间切成两半（不需要实际移动元素，只需确定下标范围） |
| **Conquer** | 递归地对左半和右半分别进行归并排序（当子数组只剩 1 个元素时自然有序） |
| **Combine** | 用 `Merge` 操作把两个已排好序的子数组合并成一个有序数组 |

**伪代码：**
```
算法 MergeSort(A, low, high)
    if low < high then
        mid ← ⌊(low + high) / 2⌋
        MergeSort(A, low, mid)       // 递归排左半
        MergeSort(A, mid+1, high)    // 递归排右半
        Merge(A, low, mid, high)     // 合并两个有序段
```

**Merge 操作**：用两个指针分别指向左右子数组的开头，每次比较指针所指元素，把较小的放入临时数组，移动相应指针——就像两叠排好的扑克牌，每次取最上面的较小那张。

**时间复杂度**：$T(n) = 2T(n/2) + O(n)$ → $\Theta(n \log n)$。

**▶ 举一反三——新例题：**

**【新题】** 简述分治法思想，并用**二分查找**说明分治法的三步骤。

**【答案】** 

| 步骤 | 二分查找中的体现 |
|------|-----------------|
| **Divide** | 将搜索区间分为 `[left, mid-1]` 和 `[mid+1, right]` 两半 |
| **Conquer** | 比较 `a[mid]` 与 x，确定在左半还是右半搜索（只搜一边！） |
| **Combine** | 二分查找不需要合并步骤——左右两边的结果直接就是最终结果 |

**【简略解析】** 二分查找是分治法的特殊情况——只需治一边（所以叫"减治法"更准确）。这解释了为什么它的复杂度是 $\Theta(\lg n)$ 而不是 $\Theta(n \log n)$。

### 2. 叙述快速排序算法的过程，并分析其最好、最坏和平均时间复杂度（5分）

**▶ 题干：**

叙述快速排序算法的过程（最好用伪代码），并分析其最好、最坏和平均时间复杂度。

**▶ 通俗解释：**

**快速排序（QuickSort）** 也是分治法，但它的"分"靠的是一个**支点元素（pivot）**：比支点小的放左边，比支点大的放右边——这步叫"分区（Partition）"。然后递归地对左右两部分重复这个操作。

与归并排序的关键区别：快速排序的**主要工作**在"分"的阶段（Partition），几乎不需要"合"；而归并排序的工作重心在"合"的阶段（Merge）。

> 就像军训站队：班长喊一个身高数（支点），比这个矮的站左边，比这个高的站右边，然后左右分别再选班长、再喊……直到每个人都在正确的位置。

**▶ 答案：**

**伪代码：**

```
算法 QuickSort(A, low, high)
    if low < high then
        pivot_pos ← Partition(A, low, high)
        QuickSort(A, low, pivot_pos - 1)
        QuickSort(A, pivot_pos + 1, high)

算法 Partition(A, low, high)
    pivot ← A[high]                  // 选最后一个元素为支点
    i ← low - 1                      // i 指向"小值区"末尾
    for j ← low to high-1 do
        if A[j] ≤ pivot then
            i ← i + 1
            交换 A[i] 和 A[j]
    交换 A[i+1] 和 A[high]           // 支点归位
    return i + 1
```

**时间复杂度分析：**

| 情形 | 条件 | 递归方程 | 复杂度 |
|------|------|----------|--------|
| **最好** | 每次支点恰好是中位数，左右均分 | $T(n) = 2T(n/2) + O(n)$ | $\Theta(n \log n)$ |
| **最坏** | 每次支点是最大/最小值（如已排序数组取最后一个） | $T(n) = T(n-1) + O(n)$ | $\Theta(n^2)$ |
| **平均** | 支点随机 | $T(n) = \frac{2}{n}\sum_{k=1}^{n-1} T(k) + O(n)$ | $\Theta(n \log n)$ |

**最好情形详细**：分区均匀 → 递归深度 $\lg n$，每层代价 $O(n)$ → $\Theta(n \lg n)$。

**最坏情形详细**：分区极不均匀 → 递归深度 $n$，每层代价递减求和 $\sum_{i=1}^{n} O(i) = O(n^2)$。

**平均情形**：平均意义下分区大约在 $1/4$ 到 $3/4$ 之间，递归深度 $O(\lg n)$，复杂度 $\Theta(n \lg n)$。

**▶ 举一反三——新例题：**

**【新题】** 对于数组 $[8, 3, 7, 1, 9, 2]$，以最后一个元素为支点，画出一轮 Partition 的过程。

**【答案】**

- pivot = 2（最后一个元素）
- 移到最后：已经是最后一个，不用移
- 扫描过程：
  - j=0: 8 > 2，跳过；j=1: 3 > 2，跳过；j=2: 7 > 2，跳过
  - j=3: 1 ≤ 2 → i=0，交换 A[0]↔A[3] → $[1, 3, 7, 8, 9, 2]$
  - j=4: 9 > 2，跳过
- 支点归位：交换 A[1]↔A[5] → $[1, \textbf{2}, 7, 8, 9, 3]$
- 支点在位置 1，左边 1 比 2 小 ✓，右边都比 2 大 ✓

**【简略解析】** Partition 的核心不变量：循环过程中，`A[low..i]` 始终 ≤ pivot，`A[i+1..j-1]` 始终 > pivot。i 像一堵"小值围墙"的边界。

### 3. 设计 $O(n)$ 时间内选择第 $k$ 小元素作为支点的算法（5分）

**▶ 题干：**

对于快速排序算法，试设计一种能够在 $O(n)$ 时间内选择第 $k$ 小元素作为支点的算法。

**▶ 通俗解释：**

快速排序的最坏情况来源于"支点选得不好"（选了最大或最小值）。如果能总是选到**中位数**（第 $n/2$ 小的元素），那分区就一定均匀，复杂度就稳在 $O(n \log n)$。

但问题来了：怎么在 $O(n)$ 时间内**找到**中位数呢？找中位数本身听起来就像要排序——排序要 $O(n \log n)$——这不是成了"为了快速排序而先排序"的循环了？

答案：**有一种聪明的方法可以在 $O(n)$ 时间（不需要完全排序）内找到第 k 小的元素**——它就是经典的 **BFPRT 算法**（又称"中位数的中位数"算法）。

**▶ 答案：**

**算法名称**：线性时间选择算法（BFPRT / Median of Medians）

**核心思想**：用"粗糙的中位数"作为支点进行快速选择（类似快速排序的 Partition，但每次只递归一边）。

**伪代码：**

```
算法 Select(A, k)
// 在数组 A 中找到第 k 小的元素（1 ≤ k ≤ |A|）
输入：数组 A，目标第 k 小
输出：A 中第 k 小的元素

步骤：
1. 将 A 的 n 个元素分成 ⌈n/5⌉ 组，每组最多 5 个元素
   （最后一组可能不足 5 个）
2. 对每组分别排序（每组只有 5 个元素，每组排序 O(1)），
   找出每组的中位数，共 ⌈n/5⌉ 个中位数
3. 递归调用 Select 找到这些中位数的中位数，记为 M
4. 以 M 为支点，对 A 做 Partition，设 M 排在位置 q
5. 若 k = q，返回 M
   若 k < q，返回 Select(A[1:q-1], k)     // 在左边找
   若 k > q，返回 Select(A[q+1:n], k-q)   // 在右边找
```

**时间复杂度分析：**

递归方程：
$$T(n) \le T\left(\left\lceil \frac{n}{5} \right\rceil\right) + T\left(\frac{7n}{10} + 6\right) + O(n)$$

- 第 1 项：递归找"中位数的中位数"（$\lceil n/5 \rceil$ 个候选）
- 第 2 项：递归处理一边（以 M 分区后，至少可以排除 $3n/10$ 的元素，最多还剩 $7n/10$）
- 第 3 项：分组排序和分区的线性开销

可以证明：$T(n) = O(n)$（通过代入法验证）。

因此，**整体可以在 $O(n)$ 时间内选出第 k 小的元素**。

> 将这个算法嵌入快速排序：每次选支点时，调用 `Select(A[l..r], (r-l+1)/2)` 取中位数作为支点——保证最坏情况下也是 $O(n \log n)$。

**▶ 举一反三——新例题：**

**【新题】** 简述为什么用 Select 选支点后快速排序的最坏复杂度变为 $O(n \log n)$。

**【答案】**
- Select 每次在 $O(n)$ 时间内找到中位数
- 以中位数为支点，分区保证两边规模不超过 $n/2$
- 快速排序递归方程变为：$T(n) = 2T(n/2) + O(n)$（分区 $O(n)$ + 选支点 $O(n)$ = $O(n)$）
- 解得：$T(n) = \Theta(n \log n)$——最坏也是这个！

**【简略解析】** 虽然理论上可以做到最坏 $O(n \log n)$，但实践中 Select 的常数因子较大，通常直接用**随机选支点**达到**期望** $O(n \log n)$ 就够用了。

## 三、贪心法（连续背包问题）

---

### 题干综述

考虑 $0 \le x_i \le 1$（可以拿一部分物品！）而不是 $x_i \in \{0, 1\}$（只能全拿或不拿）的**连续背包问题**。一种可行的贪婪策略是：按**价值密度**（value density，即 $p_i / w_i$）非递增的顺序检查物品，若剩余容量能容下正在考察的物品，将其装入；否则，往背包中装入此物品的一部分。

### 通俗解释

**连续背包问题**也叫**分数背包问题（Fractional Knapsack）**。和 0/1 背包不同——这里的物品可以**切开来拿**！比如一袋米 10 斤，你可以只拿 3 斤，价值按比例计算。

> 想象你在自助餐厅：0/1 背包 = 每道菜只能整盘拿或不拿；连续背包 = 可以每道菜都只舀一勺。

**贪心策略**：按"**性价比**"（每单位重量的价值 $p_i/w_i$）从高到低拿。性价比高的先装满，装不下就切一部分进去，直到背包装满。这显然是最优的——就像你只有 100 块钱预算吃饭，当然先点"性价比高"的菜。

---

### 1. 计算给定实例的装入结果（5分）

**▶ 题干：**

$n = 3$, $w = [100, 10, 10]$, $p = [20, 15, 15]$, $c = 105$。上述装入方法获得的结果是什么？

**▶ 答案：**

**Step 1：计算价值密度**

| 物品 | 重量 $w_i$ | 价值 $p_i$ | 价值密度 $p_i/w_i$ | 排名 |
|------|-----------|-----------|-------------------|------|
| 1 | 100 | 20 | $0.2$ | ③ |
| 2 | 10 | 15 | $1.5$ | ① |
| 3 | 10 | 15 | $1.5$ | ①（并列） |

按价值密度非递增：**物品 2 → 物品 3 → 物品 1**。

**Step 2：按顺序装入**

- 先拿物品 2：重量 10，全拿，剩余容量 $105 - 10 = 95$，获得价值 15
- 再拿物品 3：重量 10，全拿，剩余容量 $95 - 10 = 85$，获得价值 15
- 最后拿物品 1：重量 100，剩余容量只有 85，**只能拿 $85/100 = 0.85$ 部分**
  - 获得价值：$20 \times 0.85 = 17$

**Step 3：总价值**
$$15 + 15 + 17 = 47$$

$$\boxed{\text{装入方案：物品 2 全拿，物品 3 全拿，物品 1 的 } 85\% \text{；总价值 } = 47}$$

> 注意：物品 1 拿了整个的 85/100，而不是切成 85 个小块——连续背包允许取任意比例。

---

### 2. 写出伪代码（5分）

**▶ 答案：**

```
算法 FractionalKnapsack(w[1..n], p[1..n], c)
输入：n 个物品的重量数组 w、价值数组 p，背包容量 c
输出：最大总价值和物品选择方案 x[1..n]（0 ≤ x[i] ≤ 1）

步骤：
1. 计算每个物品的价值密度 d[i] ← p[i] / w[i]
2. 按 d[i] 非递增（从大到小）排序物品
   （排序后重新编号，使得 d[1] ≥ d[2] ≥ ... ≥ d[n]）
3. for i ← 1 to n do
       x[i] ← 0                    // 初始化为不选
4. remaining ← c                   // 剩余容量
5. for i ← 1 to n do
       if w[i] ≤ remaining then
           x[i] ← 1                // 全拿
           remaining ← remaining - w[i]
       else
           x[i] ← remaining / w[i] // 拿一部分填满背包
           break                   // 背包已满，停止
6. total_value ← Σ_{i=1}^{n} x[i] · p[i]
7. return total_value, x[1..n]
```

**复杂度分析**：
- 排序：$O(n \log n)$
- 贪心选择：$O(n)$
- **总复杂度**：$O(n \log n)$

---

### 3. 证明这种贪心算法总能获得最优解（10分）

🦌：证明题我再跳捏

**▶ 答案：**

**证明（交换论证法 / 剪切-粘贴法）：**

**命题**：按价值密度 $p_i/w_i$ 非递增顺序装包的贪心算法，对于连续背包问题总能得到最优解。

**证明思路**：假设存在一个最优解，我们要证明它可以通过一系列不降低总价值的调整，变成贪心解。

**证明**：

设贪心解为 $X = (x_1, x_2, \ldots, x_n)$，任一最优解为 $Y = (y_1, y_2, \ldots, y_n)$。假设物品已按价值密度 $p_i/w_i$ 非递增排列。

我们要证明：$\sum x_i \cdot p_i \ge \sum y_i \cdot p_i$（贪心解不劣于任何解）。

**关键步骤**：

找出 $X$ 和 $Y$ 第一个不同的物品标号 $j$。即：
- 对 $i < j$：$x_i = y_i$（前面都一样）
- 对 $i = j$：$x_j \neq y_j$

根据贪心策略，$x_j$ 要么是 1（全拿），要么是某个分数（拿满背包）。无论如何，$x_j > y_j$（因为贪心会尽可能多拿性价比高的物品）。

现在我们要"纠正" Y 使其更接近 X 而总价值不降：

由于 $\sum x_i \cdot w_i = \sum y_i \cdot w_i = c$（背包装满），且 $x_j > y_j$，则必然存在某个 $k > j$ 使得 $y_k > x_k$（Y 在某个低性价比物品上多拿了）。

将 Y 中的物品 k 减少一个微小量 $\delta$ 的重量（价值减少 $\delta \cdot (p_k/w_k)$），同时将物品 j 增加等量 $\delta$ 的重量（价值增加 $\delta \cdot (p_j/w_j)$）。

由于 $j < k$，$p_j/w_j \ge p_k/w_k$（物品按价值密度非递增排列），所以：
$$\text{价值变化} = \delta \cdot \frac{p_j}{w_j} - \delta \cdot \frac{p_k}{w_k} \ge 0$$

即这个交换**不会降低总价值**。反复进行这种调整，最终 Y 变为 X，且每一步总价值都不降。

因此，贪心解 $X$ 的总价值 $\ge$ 最优解 $Y$ 的总价值。又 $Y$ 是最优解，所以 $X$ 的价值 $=$ $Y$ 的价值，贪心解就是最优解之一。∎

**▶ 举一反三——新例题：**

**【新题】** 判断正误并解释：如果把连续背包问题的贪心策略（按价值密度贪心）**直接**用在 0/1 背包问题上是否也能得到最优解？

**【答案】** **错误。** 反例：$w = [2, 3]$, $p = [3, 4]$, $c = 4$。
- 价值密度：物品 1 = $1.5$，物品 2 = $1.33$
- 贪心：先拿物品 1（重量 2，价值 3），剩余容量 2，装不下物品 2 → 总价值 3
- 最优：拿物品 2（重量 3，价值 4）→ 总价值 4 > 3!

贪心策略在 0/1 背包中不生效的原因是：物品不可分割，拿高性价比物品可能"浪费"剩余容量。而连续背包中不存在浪费。

**【简略解析】** 连续背包中贪心有效，是因为可以"填满空隙"；0/1 背包中贪心失效，是因为"不可分割"导致了离散约束。

## 四、动态规划（0/1 背包——元组法）

### 1. 写出 $g(i, x)$ 的动态规划递归关系式（10分）

**▶ 题干：**

设 $g(i, x)$ 表示物品 $1, \ldots, i$、背包容量 $x$ 的 0/1 背包问题的优化效益值。试写出 $g(i, x)$ 满足的动态规划递归关系式。

**▶ 答案：**

（与 21221 试卷第五题第 1 小题相同，这里补充更详细的推导）

$$g(i, x) = \begin{cases}
\max\{\, g(i-1, x),\; g(i-1, x - w_i) + p_i \,\} & \text{当 } x \ge w_i \\[8pt]
g(i-1, x) & \text{当 } x < w_i
\end{cases}$$

**边界条件**：
- $g(0, x) = 0$（对于任意 $x \ge 0$）—— 没有物品时价值为 0
- $g(i, 0) = 0$（对于任意 $i \ge 0$）—— 背包容量为 0 时价值为 0
- $g(i, x) = -\infty$（当 $x < 0$ 时）—— 容量为负是非法的

**解释**：对第 i 个物品做决策——
- **不拿**：价值 = $g(i-1, x)$（跳过它，状态不变）
- **拿**：价值 = $g(i-1, x-w_i) + p_i$（容量减 $w_i$，价值加 $p_i$）
- 取两者中**较大值**

## 五、回溯与分支-限界（最小罚款额调度）

---

### 题目

每个作业用三元组 $(p_i,d_i,t_i)$ 表示：

- $p_i$：如果这个作业没有按时完成，要交的罚款。
- $d_i$：截止期。
- $t_i$：完成这个作业需要的时间。

本题有 4 个作业：

| 作业 | $p_i$ | $d_i$ | $t_i$ |
|------|------|------|------|
| $J_1$ | 8 | 2 | 1 |
| $J_2$ | 4 | 2 | 1 |
| $J_3$ | 9 | 3 | 2 |
| $J_4$ | 2 | 1 | 1 |

目标：选择一部分作业按时完成，没做的作业交罚款，使总罚款最小。

---

### 1. 限界函数

***🦌：LC = least cost，某种程度上就是 LB = 当前代价 + 理想状态下之后的代价，然后因为是 BFS，所以每次根据这个 LB 来选全村最好的剑给勇士去挑战魔王。***

用 LC-分支限界法时，每个结点表示一个“部分决策”：

- $A$：已经决定要做的作业集合。
- $R$：已经决定不做的作业集合。
- $U$：还没决定的作业集合。

本题的下界函数取：

$$
\boxed{
lb =
\begin{cases}
\sum\limits_{i \in R} p_i, & A \text{ 可行} \\
+\infty, & A \text{ 不可行}
\end{cases}}
$$

意思是：

- 已经决定不做的作业，罚款一定要交，所以必须算进下界。
- 还没决定的作业，先乐观地认为以后都能按时完成，所以暂时不加罚款。
- 如果已经决定要做的作业根本排不出可行顺序，这个结点直接剪枝。

剪枝规则：

```text
若 lb >= best，则剪枝。
```

其中 $best$ 是当前已经找到的最小完整罚款。

---

### 2. 可行性检查方法

判断 $A$ 中的作业能不能全部按时完成，用这个简单方法：

1. 把 $A$ 中的作业按截止期 $d_i$ 从小到大排序。
2. 从前往后累加执行时间。
3. 如果每个作业完成时刻都不超过它的截止期，则可行；否则不可行。

例如 $A=\{J_1,J_3\}$：

| 顺序 | 作业 | 执行时间 | 完成时刻 | 截止期 |
|------|------|------|------|------|
| 1 | $J_1$ | 1 | 1 | 2 |
| 2 | $J_3$ | 2 | 3 | 3 |

两个作业都没有超过截止期，所以 $A=\{J_1,J_3\}$ 可行。

---

### 3. 分支顺序

为了尽快找到好解，按罚款从大到小考虑作业：

```text
J3 -> J1 -> J2 -> J4
```

注意：这里的顺序只是“决策顺序”，不是最终执行顺序。最终执行时仍然要按截止期检查。

每个作业有两个分支：

```text
做这个作业
跳过这个作业
```

---

### 4. 状态空间树

记号说明：

```text
A：决定要做的作业集合
R：决定不做的作业集合
lb：当前结点下界，也就是 R 中作业的罚款和
```

初始：

```text
A = {}
R = {}
lb = 0
best = +∞
```

状态空间树如下：

```text
根结点
A={}, R={}, lb=0
│
├─ 做 J3
│  A={J3}, R={}, lb=0，可行
│  │
│  ├─ 做 J1
│  │  A={J3,J1}, R={}, lb=0，可行
│  │  │
│  │  ├─ 做 J2
│  │  │  A={J3,J1,J2}
│  │  │  按截止期排：J1, J2, J3
│  │  │  完成时刻：1, 2, 4
│  │  │  J3 的截止期是 3，4 > 3
│  │  │  不可行，剪枝
│  │  │
│  │  └─ 跳过 J2
│  │     A={J3,J1}, R={J2}, lb=4
│  │     │
│  │     ├─ 做 J4
│  │     │  A={J3,J1,J4}
│  │     │  按截止期排：J4, J1, J3
│  │     │  完成时刻：1, 2, 4
│  │     │  J3 的截止期是 3，4 > 3
│  │     │  不可行，剪枝
│  │     │
│  │     └─ 跳过 J4
│  │        A={J3,J1}, R={J2,J4}
│  │        lb = 4 + 2 = 6
│  │        得到完整解，best = 6
│  │
│  └─ 跳过 J1
│     A={J3}, R={J1}
│     lb = 8
│     因为 lb >= best，即 8 >= 6，剪枝
│
└─ 跳过 J3
   A={}, R={J3}
   lb = 9
   因为 lb >= best，即 9 >= 6，剪枝
```

---

### 5. 最优结果

最优结点为：

```text
A={J1,J3}
R={J2,J4}
```

也就是说：

- 做 $J_1$ 和 $J_3$。
- 跳过 $J_2$ 和 $J_4$。

实际执行顺序按截止期排：

```text
J1 -> J3
```

检查：

| 作业 | 执行区间 | 完成时刻 | 截止期 | 是否按时 |
|------|------|------|------|------|
| $J_1$ | 0 到 1 | 1 | 2 | 是 |
| $J_3$ | 1 到 3 | 3 | 3 | 是 |

跳过的作业是 $J_2$ 和 $J_4$，所以总罚款为：

$$
p_2+p_4=4+2=6
$$

最终答案：

$$
\boxed{\text{最小罚款额}=6}
$$

$$
\boxed{\text{最优调度：执行 }J_1,J_3\text{，跳过 }J_2,J_4}
$$

## 总结

| 考点 | 核心方法 | 本题关键 |
|------|----------|----------|
| Master 定理 | 比较 f(n) 和 $n^{\log_b a}$ | $\log_3 27=3$ 正好相等 |
